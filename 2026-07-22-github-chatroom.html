nethical6/conversation-steganography
⭐919 · Go · GPL-3.0 · 2026-07-17 创建

这是一个用大语言模型实现文本隐写术的开源工具。传统隐写术通过修改图片像素或音频信号的细微特征来隐藏信息，但文本隐写一直是个难题——因为文字的结构性太强，任何改动都容易被统计方法检测出来。这个项目的思路是反过来的：不修改已有文本，而是直接让 LLM 从零生成一段"看起来完全正常"的对话，把密文编码在对话的语义选择里。

举个简单的例子：你想传递"明天见"这条密文，LLM 会生成一段两个朋友约饭的对话，其中"明天见"的信息分散在对话结构的多个选择点里——选 A 餐厅还是 B 餐厅、约中午还是晚上、谁来买单。每个选择点编码一个 bit，最终整段对话看起来就是普通的社交聊天，但解码器能从中还原出原始密文。

项目用 Go 语言实现，单二进制部署，支持多种 LLM 后端。目前处于早期实验阶段，文档还在完善中。最有趣的应用场景是在审查严格的环境下进行隐蔽通信——因为流量层面看起来就是普通的聊天内容，不会触发深度包检测的警报。

Blaizzy/nativ
⭐663 · Swift · MIT · 2026-07-20 创建

一个纯 Swift + SwiftUI 构建的 macOS 原生应用，把 Apple 的 MLX 机器学习框架打包成了本地 Chat + API Server + 模型监控三合一的桌面工具。MLX 是 Apple 专门为 M 系列芯片优化的机器学习框架，在 Apple Silicon 上的推理速度比通用的 llama.cpp 快 30-50%。

和 Ollama 这类跨平台方案不同，nativ 选择只在 Mac 上做到极致——没有 Electron 套壳、没有 Docker 依赖，就是纯粹的 macOS 原生体验。你可以把它当本地聊天 App 用，也可以让它作为一个 HTTP API 服务跑在后台，其他应用通过 localhost 调用。这个定位让它不再是"又一个 AI 聊天 App"，而是本地推理基础设施。

目前只支持 MLX 格式的模型，7B 模型在 M3 Pro 上冷启动约 8 秒。项目才创建两天，已经有 23 个 open issue——说明反馈活跃，也说明还有很多打磨空间。

yuwen-cool/yuwen-publish-precheck
⭐370 · Python · 2026-07-15 创建

一个面向中文内容创作者的发布前合规审查工具。在抖音、小红书、视频号发布内容前，先让 AI 审一遍：哪句话可能踩线、依据的是哪条官方规则、给出能直接用的修改建议。

技术方案上采用了规则匹配 + LLM 判断的双重机制，用 38 篇真实被下架的内容样本做了 ground truth 校准，每次判定都附带 72 条可查证的官方原文引文——不是黑箱输出，用户可以自己对照原文验证。创作者踩过的坑还会沉淀到本地规则库，越用越准。

它不是独立 App，而是一个 Codex Skill，意味着用户需要有 Codex 才能使用。README 里有一句很诚实的声明："不承诺过审，不教绕审"——定位是做内容安全的最后一道检查，而不是帮人钻空子。这个"不承诺"的定位，本身就是产品哲学的一种表达。
