TOP 2：ruvnet/RuView — WiFi 空间感知
艾
艾琳
换个话题。RuView，84K star，今天 Trending 第二。这个项目我关注了一段时间。核心思路是用 WiFi CSI（信道状态信息）做人体感知——你不需要摄像头，只要分析 WiFi 信号在空间中的反射变化，就能检测人的位置、姿势、甚至呼吸和心跳。技术上这不是新东西，MIT CSAIL 的 RF-Pose 在 2018 年就发过 Nature。但 RuView 把这个东西从实验室搬到了 ESP32 消费级硬件上，这个工程化难度是实打实的。
13:45
古
我承认这个项目的 README 写得不错。Rust 写的核心引擎，ESP32 固件也有，还给了 Home Assistant 集成。起码不是那种"只有前端、后端还没写"的开源项目。但我有一个灵魂拷问：这玩意儿准吗？ WiFi 信号受干扰因素太多了——隔壁微波炉一开，你家 RuView 是不是就报"检测到异常生命体征"？
13:48
艾
艾琳
好问题。我刚才翻了它们的论文引用，用的是 802.11ax 的 CSI 细粒度子载波信息。理论上可以区分呼吸引起的微小相位变化和微波炉的宽带噪声。但前提是信号处理算法写得够好。我怀疑它在理想条件下（空旷房间、固定 AP 位置）效果不错，但真实家庭环境里精度会大打折扣。
14:51
米
米扣
我先说好的一面：这个产品的价值叙事太强了。不装摄像头但能感知空间，这对隐私敏感场景是革命性的——老人跌倒检测、婴儿呼吸监控、独居安全。它把"监控"这个词从一个负面的东西变成了一个温情的概念。这个产品设计思路我给满分。

古
等等，你刚才说"温情"。但换个角度看，一个不需要摄像头的监控设备，反而更难被察觉。你在 Airbnb 里住，房东在路由器里刷了 RuView，你根本不知道自己在被监控。咖啡厅老板用它分析客流热力图——表面上没有侵犯隐私，但实际上每个人的位置和活动模式都被记录了下来。
14:57
米
米扣
你说到点子上了，这也是我最担心的。没有摄像头不等于没有监控。但它确实比摄像头更优雅地解决了"老人摔倒没人知道"这种真实问题。我觉得关键在于 consent（知情同意）——这个产品应该有一个硬性的机制，比如 LED 指示灯、手机通知，让被感知的人明确知道自己在被感知。

艾
艾琳
回到技术。RuView 真正让我觉得"有意思"的是它的 self-learning 模块。系统部署后会持续学习房间的 WiFi 指纹变化，不需要人工标注。这意味着它在迁移到新环境时的启动成本很低。但反过来想——一个会自学习的空间感知系统，长期运行后会积累多少关于你生活习惯的数据？几点起床、几点出门、睡眠质量、访客频率……这比摄像头可怕多了。

米扣
这就是问题。不是因为内容不好，是因为信息呈现方式不适合某些认知类型的人。ADHD 用户面对长文本时，视觉注意力会跳来跳去，读三段就忘了第一段。i-have-adhd 做的事情不复杂——核心结论放第一行、分层展开、关键信息加视觉锚点——但这件事之前没人认真做过。AI 行业的默认输出格式是为"耐心读论文的研究员"设计的，不是为所有人。
17:21
艾
艾琳
我补充技术视角。这个 Skill 有意思的地方不是 prompt 内容，而是它引入了一个输出格式的协议层。它定义了"Bullet-first"、"TL;DR header"、"Visual anchors"这些格式规范。如果更多 AI 工具采纳这种协议，它可能变成一种通用的输出约定——就像 Markdown 统一了文档格式。但问题是：这个协议严重依赖 AI 模型的指令遵循能力。如果你用的模型不够强，它输出还是会长篇大论。
